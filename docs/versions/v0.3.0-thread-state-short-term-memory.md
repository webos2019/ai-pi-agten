# v0.3.0 — 多会话短期记忆 (ThreadState)

## 版本目标

- 实现当前浏览器会话的服务端短期记忆，刷新不丢失对话
- 多会话容器：每个浏览器会话拥有多个独立会话（Conversation）
- 每个会话对应一个独立的 ThreadState（三层记忆）
- 三层记忆结构：recent messages + summary + pinned decisions
- 模型上下文由服务端 ThreadState 统一构建，不信前端完整历史
- 工具/Agent 只存最终用户可见文本，执行过程不进记忆
- 长对话自动压缩，早期内容归入摘要
- 会话注册表管理当前浏览器会话下的会话索引（≤10）

## 非目标

- 不做长期记忆或向量记忆
- 不做持久化到数据库（内存存储，服务重启清除）
- 不做 Agent 执行轨迹记忆
- 不做 Memory Inspector
- 不做多浏览器会话同步

## 关键变更

1. 新增 `thread_state.py`：
   - `ThreadState` 三层记忆容器
   - `ThreadMessage` 纯文本消息（id/role/text/created_at）
   - `ThreadStore` 线程内存存储单例
   - `Conversation` 会话元数据（通过 thread_id 指向 ThreadState）
   - `ConversationRegistry` 会话注册表
   - `SessionStore` 浏览器会话存储
   - `compact_thread` 模型驱动的压缩
   - `TextCollectingWriter` 收集最终文本的流包装器
2. `app.py` 新增会话 API：
   - `GET /api/conversations?session_id=` 获取会话注册表
   - `POST /api/conversations` 创建新会话
   - `GET /api/conversations/{id}?session_id=` 获取会话详情（hydration）
   - `PATCH /api/conversations/{id}` 切换选中 / 重命名 / touch
   - `DELETE /api/conversations/{id}?session_id=` 删除会话
3. `chat_service.py` 集成 ThreadState：上下文构建 + 回合后写入 + 压缩触发
4. 三层记忆：recent（≤8 条纯文本）+ summary（≤2500 字）+ pinned（≤20 条）
5. 写入策略：只在回合完成后写一次，流级错误不写入
6. 压缩策略：超 8 条触发模型生成 summary + pinned_decisions
7. 安全降级：memory 失败不影响用户已收到的回答
8. 空白草稿：新建会话不立即持久化，首条消息后才创建正式会话
9. 服务端校验选中会话：模型上下文/写入均以服务端确认为准
10. 流式输出中禁止切换会话（防串线）

## 测试计划

- 刷新恢复：对话后刷新页面，消息从服务端恢复
- 长对话压缩：超过 8 条后触发 compaction，早期内容进入 summary
- 工具隔离：工具调用结果不进入 ThreadState，只有最终文本进入
- Agent 隔离：Agent 执行步骤不进入 ThreadState，只有最终回答进入
- 错误降级：流级错误时不写入记忆，不影响下一轮
- 会话注册表：创建/切换/重命名/删除会话
- 会话上限：超过 10 个会话裁剪最久未活跃的
- 消息去重：同一轮最多写一次
- 助手回答截断：超 8000 字做确定性截断

## 交付结果

- 刷新后当前会话能恢复，不是空白聊天
- 长对话后模型看到的是整理过的上下文，不是无限历史
- 工具和智能体的最终回答可以成为聊天记忆，但中间执行态留在运行时边界内
- 多会话管理：左侧面板可创建、切换、重命名、删除会话
