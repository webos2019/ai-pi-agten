# v0.1.0 — 受控 Tasklist Agent

## 版本目标

- 实现第一个受控 Agent：Pi Tasklist Agent
- 基于 `/tasklist` + `@docs://versions/*.md` 显式入口，二者缺一不可
- Runtime 读取版本方案 → Agent 生成任务清单草稿
- 确定性质量门校验结构完整性，区分阻断与警告
- 必要时最多自动修正一次，并评估修正效果
- 最终输出可复制草稿、校验结论、修正效果和人工确认点

## 非目标

- 不做通用 Agent（开放式 Planner 让模型自由规划步骤）
- 不做多 Agent 协作
- 不做长期记忆或 RAG / chunking / indexing
- 不自动扫描 docs/versions/ 目录、不自动发现最新版本方案
- 不自动写入 docs/tasklists/ 目录
- 不从用户目标直接生成 tasklist（必须引用版本方案）
- 不做持久化 Agent 状态（AgentState 仅本轮内存）

## 关键变更

1. Composer 支持 `/tasklist` 斜杠命令
2. 资源菜单支持 `@docs://versions/*.md` 引用
3. Chat Orchestrator 新增 Agent 受控分支（最优先短路）
4. `AgentState` 管理草稿 v1/v2、校验结果、修正次数、修正效果（仅本轮内存）
5. `resolve_version_plan_uri` 资源解析 + `list_available_version_plans` 可用方案列表
6. `extract_plan_structure` (planExtract) 提取版本方案结构化依据：
   version / goals / non_goals / key_changes / test_plan / deliverables
7. `validate_tasklist_structure` 确定性质量门，5 项检查 + severity 分流：
   - `missing_title` [blocking]
   - `missing_plan_uri` [blocking]
   - `missing_steps` [blocking 0 项 / warning 1-2 项]
   - `missing_checklist` [blocking 0 项 / warning 1 项]
   - `missing_verification` [warning]
8. Warning 分流：仅 blocking issues 触发修正，warning 直接忽略
9. 最多一次自动修正循环（`revision_count` 上限 1）
10. `_select_best_draft` 修正效果评估：比较 v1/v2 阻断数与警告数，选最优草稿
11. 最终输出包含可复制草稿 + 校验结论 + 修正效果 + 人工确认点 + 未自动写入声明

## 测试计划

- 入口检测：缺少版本方案引用时给出明确提示并列出可用方案
- 入口检测：引用不存在的版本方案时报错并列出可用方案
- 草稿生成：模型基于 planExtract 结果生成 tasklist
- 结构校验：检查标题、来源 URI、步骤数、勾选项、验证内容
- Warning 分流：warning 不触发修正，blocking 才触发
- 修正循环：v1 校验失败 → 修正 v2 → 再次校验（最多一次）
- 修正效果评估：v2 更好用 v2，越修越差回退 v1
- 最终输出：可复制草稿 + 校验结论 + 人工确认点 + 未自动写入声明
- 无 `/tasklist` 命令时不触发 Agent 路径

## 交付结果

- Pi Tasklist Agent 完整链路跑通
- Agent 边界可控（入口明确、步骤固定、停止条件清晰）
- 最终输出可接力（可复制草稿，留给开发者确认后落地）
