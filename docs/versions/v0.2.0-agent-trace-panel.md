# v0.2.0 — Agent Trace Panel 与流式协议增强

## 版本目标

- 新增 Agent 执行轨迹的可视化协议事件
- 流式协议支持 `agent_step_start` / `agent_step_end` chunk
- Agent 步骤含 run_id、step_index、actionType、title、status、summary、durationMs
- 前端可观察 Agent 执行过程（步骤折叠、资源/工具细节展示）
- 资源读取支持 `resource_start` / `resource_end` 含内容预览与截断标记

## 非目标

- 不做通用 Agent Runtime（仅 Tasklist Agent）
- 不做多 Agent 协作
- 不做长期记忆或 RAG
- 不自动写入 docs/tasklists/ 目录
- 不做 Agent 执行轨迹持久化（仅本轮流式可见）

## 关键变更

1. 流式协议新增 `agent_step_start` / `agent_step_end` chunk 类型
2. `stream/protocol.py` 新增工厂函数：`create_agent_step_start_chunk` / `create_agent_step_end_chunk`
3. Agent 步骤动作类型枚举 `AGENT_STEP_ACTIONS`：
   - `read_resource` 读取版本方案
   - `plan_extract` 提取版本方案结构
   - `plan_readiness` 方案就绪度评估
   - `draft_tasklist` 生成草稿
   - `validate_tasklist` 确定性质量门校验
   - `revise_tasklist` 自动修正（仅 blocking）
   - `revision_eval` 修正效果评估
   - `final_answer` 生成最终输出
4. 每步记录 `durationMs` 便于性能分析
5. 资源读取 `resource_end` 支持 `contentPreview` + `isTruncated` + `previewChars`
6. `stream/lifecycle.py` 统一管理流生命周期（start/done/error 幂等发射）

## 测试计划

- 验证 `/tasklist` + `@docs://versions/*.md` 入口检测
- 验证 Agent 步骤序列完整发射（step_start / step_end 配对）
- 验证每步 status 为 success / error / skipped 之一
- 验证 read_resource 失败时给出明确错误 + 可用方案列表
- 验证 plan_readiness 不足时短路返回提示
- 验证无版本方案引用时的明确提示
- 验证资源读取的 contentPreview 与 isTruncated 正确

## 交付结果

- Agent Runtime 可控链路跑通，步骤可观察
- 前端可渲染 Agent 执行过程
- 最终输出可复制 tasklist 草稿
